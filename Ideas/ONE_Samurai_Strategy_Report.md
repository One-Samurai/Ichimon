# ONE Samurai：日本市場進軍與 Web3 產品戰略提案報告

## 執行摘要 (Executive Summary)

針對 ONE Championship 於日本推出的「ONE Samurai」系列賽（5 年內舉辦 60 場大賽），我們在落地執行面臨四大核心挑戰：法規紅線、粉絲數據回收、場館成本及選手後勤。

為突破這些痛點並確保長久營運（Go-To-Market），本提案規劃四款基於 Sui Network 的創新工具。我們捨棄傳統 Web2 的封閉系統架構，利用 Sui 的物件導向（Object-Centric）特性、低延遲與高擴展性，將合規邏輯、粉絲身份、實體場地與營運資源徹底「資產化」。本戰略不僅完全規避日本嚴格的賭博與證券法規，更創造可持續的粉絲經濟價值與多方信任生態。

---

## 1. 日本市場拓展的四大核心痛點

1. **法規與行銷風險**
   - **痛點**：日本對賭博及獎品價值有嚴格規範（如 Unfair Premiums Act）。若行銷活動涉及付費且發放金錢型/高額獎勵，極易誤觸刑法及博彩法規；虛擬資產亦有被移至證券法（FIEA）監管之風險。
2. **粉絲經濟斷層**
   - **痛點**：轉播權獨家授權給 U-NEXT（千萬訂戶），導致 ONE 無法直接掌握觀眾數據。缺乏直接與粉絲溝通的渠道與數據資產，在與本土品牌（如 RIZIN）的粉絲故事行銷競爭中陷入劣勢。
3. **場地租賃與營收天花板**
   - **痛點**：每月租用有明競技場（Ariake Arena）等昂貴場館，若僅依賴賽事當晚的門票收入，難以覆蓋場租與高額維護成本，導致 ROI 極低。
4. **選手後勤與營運效率**
   - **痛點**：大量國際選手頻繁進出日本，機票、住宿、訓練設施與簽證需求複雜。仰賴內部 Excel 與 Email 往返，極易出錯並損害選手體驗，不利於建立「本土英雄」陣容。

## 2. Sui 驅動的產品提案與 GTM 策略

針對上述痛點，提出四個具備長期商務價值的解決方案，並詳述為何我們必須使用 Sui Network 而非 Web2 中心化後端。

### 提案一：Campaign Compliance Studio (合規活動設計器)
- **產品定位**：供行銷與贊助商使用的拖拉式活動建置後台，內建日本法規參數檢核。
- **GTM 策略**：讓內部行銷部門預先組合安全玩法（無收費、獎額受控），系統產出給法務自動審核的摘要。
- **為何使用 Sui 而非 Web2?**
  - **合規邏輯鏈上化**：將活動規則（Campaign Config）寫成 Sui 上可驗證的物件，保證任何發放與抽獎邏輯都受此約束，防堵人為修改。
  - **創造多方單一真相**：Web2 架構下，行銷需求、法務審查與工程實作常脫節。Sui 讓 ONE、法務與贊助商都可以直接讀取同一個合約配置。
  - **高度可組合性**：任何新的小遊戲前端都能直接調用鏈上 Campaign Object 更新額度狀態，開發快速且安全。

### 提案二：Fan Passport 2.0 (全渠道粉絲通行證)
- **產品定位**：綁定粉絲身份、成就紀錄與權益的動態 NFT（不可轉讓以避開投資商品規範）。
- **GTM 策略**：透過 U-NEXT 轉播頁面或現場 QR code 免費領取。衍生「無金流 Pick'em 預測」與「Gym 打卡 (Train Like a Samurai)」模組，提供專屬實體體驗（如見面會、提早購票）而非獎金。
- **為何使用 Sui 而非 Web2?**
  - **跨平台資產自主權**：Web2 會員被平台綁死。Sui 上的 Passport 可輕易讓 U-NEXT、官方 App 乃至線下健身房直接共用並核實粉絲身分。
  - **無摩擦的 Web3 體驗**：利用 Sui 的 `zkLogin`，粉絲能以 Google 等社交帳號無痕登入取得錢包與 NFT，完美去除教育日本大眾使用私鑰的巨大門檻。
  - **動態進化能力**：Sui 支援物件屬性隨時變更（Dynamic fields）。粉絲的打卡與預測連勝紀錄可直接更新在 NFT 特徵上，不需複雜的 Web2 API 狀態同步。

### 提案三：Samurai Arena Experiences (場館體驗碎片化市場)
- **產品定位**：將非比賽時段（彩排、後台導覽、提前包廂）切割鑄造為 Experience Pass 發售。
- **GTM 策略**：把單一賽會拉長為一整天的體驗產品，直接向高淨值粉絲或贊助商兜售，最大化場地 ROI。
- **為何使用 Sui 而非 Web2?**
  - **實體服務資產化與流通**：Web2 票務系統易成為數據孤島。Sui 可將拆分的時段鑄造成可組合的 pass，並直接設定轉讓、分潤與防黃牛邏輯。
  - **自動清算與對帳**：各合作方（場館、ONE、贊助商）的分潤可透過 Move 合約直接結算，取代 Web2 冗長耗時的跨組織人工對帳。
  - **即時的高併發狀態處理**：Sui 的低延遲保證名額控管與現場 QR 驗票能做到毫秒級同步，防止超賣。

### 提案四：Samurai Fighter Hub (在日後勤共享 OS)
- **產品定位**：管理選手在日行程、偏好與設施配對的 B2B 資源網。
- **GTM 策略**：先供 ONE 內部與經紀人使用，打通行程確認環節。個資放於鏈下依循日本 APPI 法規，鏈上僅存留去識別化 ID 與資源物件。
- **為何使用 Sui 而非 Web2?**
  - **跨界信任層**：後勤協調牽涉 ONE 官方、日本道館與外部機構。Sui 提供的共享狀態層可破除各自系統的權限迷宮，安全共享排程。
  - **稀缺資源的防呆鎖定**：運用 Move 語言在語言層面防範物件雙化的特質，天然保證「特定時段的訓練場預約資源」不會被重複預訂。
  - **開放後續生態延伸**：當資料（如無敏的到訪紀錄）皆依循 Sui 的標準物件規格儲存後，未來能隨時開放授權給粉絲產品（如道館巡禮任務）無縫調用。

## 3. 執行建議（Steering Committee 決策參照）

以在本次 Hackathon 迅速展示 MVP 及確保最低法規風險為考量，**建議由「Fan Passport 2.0 整合無金流賽事預測模組」切入**：
1. **業務衝擊最大**：從 U-NEXT 將公域流量轉私域資產，精準打中行銷痛點。
2. **合規防護最穩**：主打免費領取與體驗型獎勵，避開加密貨幣法與博弈法規範。
3. **技術火力展示**：完整展現 Sui 的 `zkLogin`（無阻礙開戶）、`Dynamic NFT`（隨戰績成長的資產）及高頻低延遲（賽事即時預測）三大王牌優勢。
